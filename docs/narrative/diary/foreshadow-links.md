## 1. Princípio

[[foreshadowing]] cataloga 130 plants distribuídos por canais diversos: prop ambiental, fala casual de NPC, doc descobrível, beat cômico LucasArts, vinheta sensorial, mecânica de stats degradada, áudio diegético, glitch visual ambient. Nem todo plant vai pro Diário. Plants vão pro Diário quando:

1. **Gus observa diretamente e anota** → entry tipo Manuscrito (cursiva 11 anos, [[entries-manuscrito-glossario]]).
2. **O plant é um doc descobrível** → entry tipo Pickup catalog (espelha [[in-world-docs]], catalogado em [[entries-docs-descobriveis]]).
3. **O plant é um detalhe pessoal de NPC / facção** → entry tipo Ficha (atualização incremental, [[entries-fichas-bestiary]]).
4. **O plant é um landmark / evento cronológico** → entry tipo Mapa ou Timeline (auto-gerado conforme exploração ou beat canônico, [[entries-mapas-timeline]]).

Plants que existem apenas como **ambient** (áudio, mood, glitch shader, propaganda em loop, prop sem leitura ativa do Gus) **não viram entry do Diário**. Ficam no mundo; player percebe sem o Diário registrar. Exemplos: F033 (tela LED travada Janelarum), F042 (loop "continue continue continue"), F048 (zonas deleção GRE — visualmente óbvias mas Gus não anota "vi mais uma clareira cinza" toda vez). Categorizados explicitamente como **AMBIENT ONLY** na tabela §2.

Knowledge alta destrava 2ª-passada hints: o Diário "lembra" plants iniciais com novo contexto. **Sticky note visual** (post-it diegético colorido) marca entry com revelação retroativa disponível. Player clica/hover → texto adicional revelado. Cores canônicas em §3.

Cross-link Bloco I é **pesado por design** (decisão macro Bloco H §7): cada plant relevante aponta entry-setup + entry-payoff. Knowledge alta destrava sticky-notes que conectam visualmente plant → payoff dentro do Diário — player vê linha de costura entre as páginas. Função: recompensar investigação cumulativa; honrar Chekhov rigoroso ([[foreshadowing]] §Princípio).

## 2. Tabela mestre — 130 plants × Diário

Convenções de colunas:

- **F###** — ID do plant em [[foreshadowing]].
- **Categoria** — abreviação canônica (DAN=Dante, STE=Sterling, PAT=Patch-Zero, COM-X=Companion X, E1/E2/E3=Era 1/2/3, END=Ending gate, MEC=Mecânico).
- **Tipo Diário** — Manuscrito / Pickup / Ficha / Mapa / Timeline / **AMBIENT ONLY**.
- **Entry ID Diário** — identificador interno (convenção: M-### para Manuscrito; P-DOC## para Pickup doc in-world; F-XXX-## para Ficha personagem/facção; MAP-SS-## para Mapa setting; T-### para Timeline). IDs canônicos definitivos a finalizar em produção; os listados são placeholders consistentes.
- **Knowledge gate (2ª-passada)** — gate que destrava sticky note conectiva ou texto adicional.
- **Sticky note** — cor (ver §3) ou "—" se não aplicável.
- **Notas operacionais** — instruções específicas pro Diário (e.g. atualização double-layer pós-reveal, cross-link manual disponível).

## 3. Sticky notes (2ª-passada hints)

### 3.1 Como funcionam

Sticky note diegética é **marcador visual no Diário** (in-universe: post-it colorido colado pelo Gus quando ele percebe conexão entre páginas). Aparece em entries quando Knowledge atinge gate específico, sinalizando "esta entry tem mais a oferecer". Texto adicional revelado em hover/click. Pode aparecer múltipla (vermelha + amarela + azul em entry crítica de Ouro Dante-companion-ending).

Visualmente: post-it colorido pequeno sobre a borda da página, com rabisco de letra Gus indicando categoria ("liga com [F###]"; "ver [setting]"; "Mariana → Dante?"). Clique abre painel lateral com cross-ref expandido. Não invade conteúdo principal da página.

**Sticky note não aparece se Knowledge gate não foi atingido.** Mesmo player que revisita uma entry 50 vezes não vê a sticky se categoria correspondente está zerada. Crescimento é orgânico ao progresso real.

### 3.2 Cores canônicas

- **Vermelha** — Dante traidor foreshadow. Cresce ao longo do jogo conforme stats Gus degradam (F013) e plants Dante são colhidos. Pós-reveal climax Etapa 1, todas as sticky vermelhas ficam **ativas permanentemente** (não somem). Player pode revisitar e ler em sequência cronológica todas as pistas que perdeu.
- **Cinza-prateado** — Sterling / Patch-Zero conexão profunda. Knowledge média/alta. Cresce com posse de docs 1, 5, 12, 14 e leitura de fichas Sterling-Penedo.
- **Verde** — Lore Era 1 / cripto-glifo decifrado / preservação ancestral. Knowledge alta com Bento na party multiplica eficiência.
- **Amarela** — Companion arc payoff. Específica do companion correspondente. Knowledge média na categoria Companion-X.
- **Azul** — Endings gates. Gate Bronze / Prata / Ouro. Mais escassa; só aparece em plants críticos de ending.

### 3.3 Sticky note múltipla

Plants críticos com cross-categoria (ex: F021 caixa zinco Galpão Cooperativo = **vermelha + azul**; F128 Beatriz Pólvora epilogue = **amarela + azul**; F074 caixa zinco Inácia = **amarela + vermelha + verde**) recebem múltiplas sticky stackedas visualmente. Player vê 3 post-its sobrepostos em uma entry — sinal de cross-link denso disponível.

### 3.4 Sticky de cross-reference manual (Mecânico-meta)

## 9. Anti-patterns padrão (checklist de rejeição)

Aplicáveis a foreshadow-link Diário:

- **Plant frouxo (Chekhov não-pago).** Reprovado: auditoria §7 exige cobertura 100%.
- **Payoff sem plant.** Reprovado: cada reveal canônico tem plant distribuído em pelo menos um canal.
- **Sticky note sem critério.** Reprovado: sticky aparece sob gate específico, não arbitrariamente.
- **Spoiler antecipado via sticky.** Reprovado: sticky só aparece quando Knowledge categoria atinge gate; player Bronze nunca vê sticky vermelha pós-reveal antes de cruzar campanha.
- **Lore dump em entry.** Reprovado: entry curta, voice Gus, sem tratado.
- **Mistura de tipo de entry.** Reprovado: cada plant mapeia para tipo canônico ([[_INDEX]] §97).
- **Em-dash horizontal "—" em narrativa.** Reprovado (anti-pattern global CLAUDE.md raiz).
- **Adultos como autores ocultos do Diário.** Reprovado: Diário é do Gus. Outros NPCs aparecem só via documentos descobríveis.
- **Bypass de sticky via UI.** Reprovado: sticky respeita gate sempre. Sem cheat.
- **Cross-reference automática default.** Reprovado: cross-ref manual é opt-in; opt-in mode "Auto-cross-reference" existe como acessibilidade, +1 ao invés de +2.
